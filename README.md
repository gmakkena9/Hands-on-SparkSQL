# Hands-on SparkSQL — Social Media Sentiment Analysis

A PySpark / Spark SQL assignment that analyzes a synthetic social-media dataset
(`posts.csv` + `users.csv`) across four tasks and writes the results to CSV.

## Prerequisites

| Software   | Version needed                | Notes                                            |
| ---------- | ----------------------------- | ------------------------------------------------ |
| Python     | 3.x                           | `python --version`                               |
| **Java**   | **17 or 21 (NOT 24/25)**      | Spark 4.x does not run on Java 24/25 yet         |
| PySpark    | 4.x                           | `pip install pyspark`                            |
| Git        | any                           | only needed to clone / submit                    |

> **Critical:** Spark 4.x must run on **Java 17 or 21**. Newer JDKs (24, 25)
> throw `UnsupportedOperationException: getSubject is not supported` because
> Spark's bundled Hadoop calls a security API that those JDKs disabled.
> Install **Eclipse Temurin JDK 17** if you are on a newer Java.

## Setup Steps

### 1. Install PySpark

```cmd
pip install pyspark
```

### 2. Install Java 17 (if you are on Java 24/25)

Download **Temurin JDK 17** (Windows, x64, JDK, `.msi`) from:
https://adoptium.net/temurin/releases/?version=17

Install with defaults. It lands in something like:
`C:\Program Files\Eclipse Adoptium\jdk-17.x.x.x-hotspot`

Find the exact folder name:

```cmd
dir "C:\Program Files\Eclipse Adoptium"
```

### 3. Point your terminal at Java 17

Run these in the SAME terminal window (substitute the real folder name).
**These settings only last for the current window** — redo them in any new window.

```cmd
set JAVA_HOME=C:\Program Files\Eclipse Adoptium\jdk-17.0.19.10-hotspot
set PATH=%JAVA_HOME%\bin;%PATH%
java -version
```

`java -version` MUST print `version "17..."` before continuing.

### 4. Clone the project and enter it

```cmd
git clone https://github.com/gmakkena9/Hands-on-SparkSQL.git
cd Hands-on-SparkSQL
```

### 5. Create the data/output folders and generate the dataset

```cmd
mkdir input
mkdir outputs
python input_generater.py
```

This creates `input/users.csv` (50 users) and `input/posts.csv` (100 posts).
The data is **random**, so your numbers will differ from any example output.

## Running the Tasks

Run each script with `python` (no `spark-submit` needed — PySpark from pip
includes its own Spark):

```cmd
python src\task1.py
python src\task2.py
python src\task3.py
python src\task4.py
```

Each script prints a results table to the screen and writes a CSV into `outputs/`.

Verify all four outputs were created:

```cmd
dir outputs
type outputs\hashtag_trends.csv
type outputs\engagement_by_age.csv
type outputs\sentiment_engagement.csv
type outputs\top_verified_users.csv
```

## What Each Task Does

**Task 1 — Hashtag Trends** (`hashtag_trends.csv`)
Splits the comma-separated `Hashtags` column into individual tags, counts how
often each appears, and sorts descending. Columns: `Hashtag, Count`.

**Task 2 — Engagement by Age Group** (`engagement_by_age.csv`)
Joins posts to users on `UserID`, groups by `AgeGroup`, and computes average
likes and retweets per group. Columns: `AgeGroup, Avg_Likes, Avg_Retweets`.

**Task 3 — Sentiment vs Engagement** (`sentiment_engagement.csv`)
Labels each post by `SentimentScore` (Positive > 0.3, Negative < -0.3, else
Neutral), then averages engagement per label.
Columns: `Sentiment, Avg_Likes, Avg_Retweets`.

**Task 4 — Top Verified Users by Reach** (`top_verified_users.csv`)
Filters to verified users, joins their posts, defines reach as
`Likes + Retweets`, sums reach per user, and returns the top 5.
Columns: `Username, Total_Reach`.

## Windows-Specific Notes (Common Errors & Fixes)

**`getSubject is not supported`** — You are on Java 24/25. Install Java 17 and
re-point `JAVA_HOME` (Setup Steps 2–3).

**`HADOOP_HOME and hadoop.home.dir are unset` / `winutils.exe`** — Spark's
native CSV writer needs `winutils.exe` on Windows. This project avoids that by
writing output with pandas instead:

```python
df.toPandas().to_csv("outputs/<name>.csv", index=False)
```

(This is fine for small datasets. For large data, install `winutils.exe`,
set `HADOOP_HOME`, and use Spark's `.write.csv(...)`.)

**`PermissionError: ... Permission denied: 'outputs/<name>.csv'`** — A leftover
directory (from an earlier failed Spark write) is blocking the file. Delete it
and re-run:

```cmd
rmdir /s /q outputs\hashtag_trends.csv
```

**The `winutils` WARN line still appears even when it works** — That is a
harmless warning, not an error, once output is written via pandas. Ignore it.
